1. **Solution structure** — run these commands from project root:
   ```
   dotnet new sln -n EquipmentManagement -o .
   dotnet new classlib -n KhoThietBi.Domain -o src/KhoThietBi.Domain --framework net9.0
   dotnet new classlib -n KhoThietBi.Application -o src/KhoThietBi.Application --framework net9.0

   ```

   - **KhoThietBi.Infrastructure**: `Npgsql.EntityFrameworkCore.PostgreSQL` 9.x, `Microsoft.EntityFrameworkCore.Tools` 9.x, `Microsoft.AspNetCore.Identity.EntityFrameworkCore` 9.x, `Serilog.AspNetCore` 8.x, `Serilog.Sinks.File`
   - **KhoThietBi.API**: `Microsoft.AspNetCore.Authentication.JwtBearer` 9.x, `Swashbuckle.AspNetCore` 7.x (or Scalar)

   `BaseEntity.cs`:
   ```csharp
   public abstract class BaseEntity
   {
       public Guid Id { get; set; }
       public DateTime CreatedAt { get; set; } = DateTime.UtcNow;
       public DateTime? UpdatedAt { get; set; }
   }
   ```

   `Category.cs` — inherits BaseEntity. Properties: `string Name` (required, max 100), `string? Description` (max 500). Navigation: `ICollection<Equipment> Equipment` (empty collection init — Equipment entity added in Phase 2).

   `Supplier.cs` — inherits BaseEntity. Properties: `string Name` (required, max 200), `string? Address` (max 500), `string? ContactInfo` (max 200).

   `Department.cs` — inherits BaseEntity. Properties: `string Name` (required, max 100).

   `AuditLog.cs` (per D-11) — Properties: `Guid Id`, `string UserId`, `string Action` (Create/Update/Delete), `string EntityType`, `string EntityId`, `DateTime Timestamp` (UTC), `string? OldValues` (JSON), `string? NewValues` (JSON). Do NOT inherit BaseEntity — AuditLog has its own schema.

   `PagedResult<T>` generic class: Properties `List<T> Items`, `int TotalCount`, `int Page`, `int PageSize`. Computed property `int TotalPages => (int)Math.Ceiling(TotalCount / (double)PageSize)`.

5. **Application layer** (per D-04 feature-based folders, per D-05 MediatR):

   `IApplicationDbContext.cs` interface: exposes `DbSet<Category>`, `DbSet<Supplier>`, `DbSet<Department>`, `DbSet<AuditLog>`, and `Task<int> SaveChangesAsync(CancellationToken)`.

   `ValidationBehavior.cs`: MediatR `IPipelineBehavior<TRequest, TResponse>` that runs all registered `IValidator<TRequest>` before handler execution. On validation failure, throw a custom `ValidationException` with the error list.

   `LoggingBehavior.cs`: MediatR pipeline behavior that logs request type name, start time, end time, and elapsed milliseconds using `ILogger<LoggingBehavior>`.

   `DependencyInjection.cs` with `AddApplication(this IServiceCollection services)` extension: registers MediatR from the Application assembly, registers FluentValidation validators from the assembly, registers AutoMapper profiles from the assembly, registers pipeline behaviors (ValidationBehavior, LoggingBehavior).

   `DependencyInjection.cs` with `AddInfrastructure(this IServiceCollection services, IConfiguration configuration)`: registers `ApplicationDbContext` with PostgreSQL connection string from `configuration.GetConnectionString("DefaultConnection")`, registers Identity services.
