**Step 2 â€” Create CustomTokenController for Vietnamese error messages (D-10):**

Create file `KhoThietBi/src/KhoThietBi.HttpApi.Host/Controllers/CustomTokenController.cs`:

```csharp
using System;
using System.Threading.Tasks;
using Microsoft.AspNetCore.Mvc;
using OpenIddict.Abstractions;
using OpenIddict.Server.AspNetCore;
using Volo.Abp.DependencyInjection;
using Volo.Abp.Identity;
using Volo.Abp.OpenIddict.Controllers;
using Microsoft.AspNetCore;
using Microsoft.AspNetCore.Authentication;

namespace KhoThietBi.Controllers;

[Dependency(ReplaceServices = true)]
[ExposeServices(typeof(TokenController))]
public class CustomTokenController : TokenController
{
    private readonly IdentityUserManager _userManager;

    public CustomTokenController(IdentityUserManager userManager)
    {
        _userManager = userManager;
    }

    [HttpPost("connect/token")]
    public override async Task<IActionResult> HandleAsync()
    {
        var request = HttpContext.GetOpenIddictServerRequest();
        if (request == null || !request.IsPasswordGrantType())
        {
            return await base.HandleAsync();
        }

        // Check if user exists first
        var user = await _userManager.FindByNameAsync(request.Username);
        if (user == null)
        {
            var properties = new AuthenticationProperties(new Dictionary<string, string>
            {
                [OpenIddictServerAspNetCoreConstants.Properties.Error] = OpenIddictConstants.Errors.InvalidGrant,
                [OpenIddictServerAspNetCoreConstants.Properties.ErrorDescription] = "Tai khoan khong ton tai. Vui long kiem tra lai ten dang nhap."
            });
            return Forbid(properties, OpenIddictServerAspNetCoreDefaults.AuthenticationScheme);
        }

        // Check password
        var passwordValid = await _userManager.CheckPasswordAsync(user, request.Password);
        if (!passwordValid)
        {
            var properties = new AuthenticationProperties(new Dictionary<string, string>
            {
                [OpenIddictServerAspNetCoreConstants.Properties.Error] = OpenIddictConstants.Errors.InvalidGrant,
                [OpenIddictServerAspNetCoreConstants.Properties.ErrorDescription] = "Mat khau khong dung. Vui long thu lai."
            });
            return Forbid(properties, OpenIddictServerAspNetCoreDefaults.AuthenticationScheme);
        }

        // Valid credentials â€” delegate to base for standard token generation
        return await base.HandleAsync();
    }
}
```

**IMPORTANT NOTES on CustomTokenController:**
- The `[Dependency(ReplaceServices = true)]` + `[ExposeServices(typeof(TokenController))]` attributes are ABP's service replacement pattern
- Error messages use ASCII-safe Vietnamese (no diacritics in error_description) to avoid encoding issues in OAuth2 responses. The Angular frontend maps these to full Vietnamese display strings.
- The `HandleAsync` method checks user existence first, then password, returning distinct `error_description` values
- For valid credentials, it delegates to `base.HandleAsync()` which handles the full OpenIddict token generation

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Angular SPA -> ABP API | Untrusted HTTP requests from browser to backend API |
| /connect/token endpoint | Authentication boundary: accepts username/password, returns JWT |
| JWT token in localStorage | Client-side token storage accessible to JavaScript |
| PostgreSQL connection | Backend to database, trusted internal network |

## STRIDE Threat Register

| Threat ID | Category | Component | Severity | Disposition | Mitigation Plan |
|-----------|----------|-----------|----------|-------------|-----------------|
| T-1-01 | Spoofing | /connect/token (brute force) | HIGH | mitigate | ABP Identity lockout is pre-configured (5 failed attempts = 5 min lockout). Verify `AbpIdentityOptions.Lockout` settings in KhoThietBiHttpApiHostModule.cs. No additional config needed â€” ABP default is sufficient for ASVS 2.1. |
| T-1-02 | Information Disclosure | /connect/token (user enumeration) | MEDIUM | accept | D-10 explicitly requires distinguishing "user not found" vs "wrong password". This enables user enumeration. Accepted because: admin-only internal tool, no public registration, limited attack surface. Document this tradeoff. |
| T-1-03 | Tampering | AbpAuditLogs table | MEDIUM | mitigate | Audit log tables are append-only at application level (ABP never deletes/updates audit records). Database-level protection via PostgreSQL `pg_hba.conf` restricting direct DB access. Application user should not have TRUNCATE/DELETE on audit tables in production. For v1 dev, accepted as-is. |
| T-1-04 | Elevation of Privilege | Unauthenticated API access | HIGH | mitigate | ABP applies `[Authorize]` by default on all auto-generated API controllers. Verify by calling any /api/ endpoint without token â€” must return 401. AuthGuard on Angular side (Plan 02) provides defense-in-depth. |
| T-1-05 | Information Disclosure | JWT in localStorage (XSS) | MEDIUM | accept | D-09 locks localStorage storage. XSS could steal token. Mitigations: Angular's built-in XSS protection (template sanitization), PrimeNG sanitizes inputs, CSP headers (future). Accepted for admin-only internal tool per D-09. |
| T-1-06 | Tampering | SQL injection via login inputs | HIGH | mitigate | ABP Identity uses parameterized queries via EF Core. OpenIddict validates input format. No raw SQL in login flow. FluentValidation pipeline active. ASVS 5.3 satisfied by framework-level protection. |
| T-1-07 | Repudiation | Admin denies performing action | LOW | mitigate | ABP Audit Logging with AddAllEntities() records userId, action, timestamp, old/new values for every write operation. Satisfies ASVS 7.1 audit trail requirement. |
| T-1-08 | Spoofing | JWT token replay | MEDIUM | accept | 8-hour token lifetime (D-08) limits replay window. No refresh token in v1. For admin-only internal tool on trusted network, accepted. Future improvement: add token revocation or shorter lifetime. |
</threat_model>
