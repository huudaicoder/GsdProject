# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Admin Login + JWT + Audit.
> Covers login page, app shell, collapsible sidebar navigation, dashboard placeholder (KPI cards), and session expiry UX.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, table cell icon spacing |
| sm | 8px | Button icon gap, input inner padding-top/bottom, tight row gaps |
| md | 16px | Default element spacing, card inner padding, form field gap |
| lg | 24px | Section padding, dialog content padding, table header padding |
| xl | 32px | Layout column gap (sidebar-to-content), card-to-card gap |
| 2xl | 48px | Login panel vertical padding, KPI card grid gap |
| 3xl | 64px | Login page full-height centering offset |

Exceptions:
- Touch targets on action icon buttons (edit, delete in table rows): minimum 40px height — use `p-button-sm` with explicit `min-height: 40px`.
- Sidebar width: 256px (fixed, not on 8-point scale — PrimeNG standard sidebar width).
- Header height: 56px (PrimeNG `p-toolbar` default — not adjusted).
- KPI card height: 120px (fixed height for visual consistency across 5 cards).

## Color

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) | `#F8F9FA` | App background, page content area, table row backgrounds |
| Secondary (30%) | `#FFFFFF` | Sidebar background, cards, dialog panels, header background, KPI cards |
| Accent (10%) | `#2563EB` | Primary CTA buttons only, active sidebar item indicator, pagination current-page highlight |
| Destructive | `#DC2626` | Delete action buttons and confirmation dialogs only |

Accent reserved for:
1. "Thêm mới" (Add) button on each reference data page — one per page
2. "Lưu [tên thực thể]" (Save) button inside Add/Edit modal dialogs — one per dialog
3. "Đăng nhập" submit button on login page
4. Active sidebar navigation item left-border indicator (3px left border, background `#EFF6FF`)
5. Current page number button in table pagination
6. No other elements use accent color

Sidebar disabled items (unbuilt phases): text color `#9CA3AF` (gray-400), no hover state, `cursor: not-allowed`, `pointer-events: none`.

Login left panel background: `#1E3A5F` (deep navy) — visual contrast from the white right panel, establishes branding. System name and logo render in `#FFFFFF` on this panel.

KPI card icon color: `#2563EB` — one icon per card, 24px, uses accent but serves as data illustration not CTA.

**Source:** Claude's Discretion (CONTEXT.md — "color scheme / branding — để Claude chọn"). Professional admin tool palette using PrimeNG Lara Light Blue theme as base, overriding primary to `#2563EB`.

## Component Inventory

### 1. Login Page (`/login`)

**Layout:** Full-viewport split, two equal columns (50% / 50%).

Left panel:
- Background: `#1E3A5F`
- Content centered vertically: company logo placeholder (80x80px, white border, `#FFFFFF` text initials "KTB") + system name "Quản lý Kho Thiết Bị" at Display size + tagline "Hệ thống quản lý tài sản công ty" at Body size, color `#93C5FD`
- No interactive elements

Right panel:
- Background: `#FFFFFF`
- Content centered vertically, max-width 400px, horizontal padding 48px
- Form title: "Đăng nhập" at Heading size
- Fields: `p-inputtext` for Username, `p-password` for Password (toggle show/hide via PrimeNG built-in toggle button)
- Submit button: "Đăng nhập" — full width, `p-button` with accent color, height 44px
- Error message zone: below the submit button, rendered as `p-message severity="error"` — visible only when error state is active; see Copywriting Contract for specific messages
- No "forgot password" link (out of scope — AUTH-V2-01 deferred to v2)

Responsive: Below 768px, left panel collapses to hidden; right panel takes full width.

### 4. Dashboard Placeholder (`/dashboard`)

### 5. Session Expiry UX

When any API call returns HTTP 401 (after a valid session):
1. Angular HTTP interceptor catches the 401 response
2. Immediately displays `p-toast severity="warn"` positioned top-right
3. Toast message: "Phiên đăng nhập đã hết hạn. Vui lòng đăng nhập lại." (see Copywriting Contract)
4. Toast is NOT auto-dismissible — it stays visible until redirect completes
5. After 3 seconds, interceptor: clears token from localStorage, navigates to `/login`
6. On the login page, no additional message shown (toast from previous page is gone; the redirect itself communicates the reason)

**401 vs 403:** The interceptor distinguishes 401 (session expired → redirect to login) from 403 (forbidden → `p-toast severity="error"` "Bạn không có quyền thực hiện thao tác này." without redirect).

**Source:** CONTEXT.md D-35.

## Interaction States

| State | Pattern |
|-------|---------|
| Loading (table data) | `p-skeleton` inline rows — no full-page spinner |
| Loading (dialog save) | Save button shows `pi-spinner` icon spinning + disabled; text changes to "Đang lưu..." |
| Loading (login submit) | "Đăng nhập" button shows `pi-spinner` + disabled; text changes to "Đang xử lý..." |
| Success feedback | `p-toast severity="success"` top-right, auto-dismiss 3s |
| Error feedback (API) | `p-toast severity="error"` top-right, auto-dismiss 6s |
| Error feedback (login) | `p-message severity="error"` inline below submit button, persistent until next submit attempt |
| Form invalid — blur | `p-message severity="error"` beneath invalid field |
| Form invalid — submit | All invalid fields show errors simultaneously; focus moves to first invalid field |
| Delete confirmation | `p-confirmDialog` modal — requires explicit user confirmation |
| Delete blocked | `p-toast severity="error"` auto-dismiss 6s — no confirmation dialog shown |
| Session expired (401) | `p-toast severity="warn"` top-right, non-dismissible; redirect after 3s |
| 403 Forbidden | `p-toast severity="error"` top-right, auto-dismiss 6s; no redirect |
| Disabled nav item hover | No visual change; tooltip "Chức năng này chưa được triển khai." appears after 500ms delay |
| Sidebar group expand | Angular `[@slideDown]` animation (150ms ease-out); chevron rotates 90deg |
| Sidebar group collapse | Reverse of expand (150ms ease-in) |
| Active nav group auto-open | On route change, the group containing the active route expands without animation (instant) |
