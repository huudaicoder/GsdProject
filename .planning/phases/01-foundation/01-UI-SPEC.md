# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1: Foundation.
> Covers login page, app shell, sidebar navigation, and reference data management (Categories, Suppliers, Departments).
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.

## Spacing Scale

Declared values (all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline badge padding, table cell icon spacing |
| sm | 8px | Button icon gap, input inner padding-top/bottom, tight row gaps |
| md | 16px | Default element spacing, card inner padding, form field gap |
| lg | 24px | Section padding, dialog content padding, table header padding |
| xl | 32px | Layout column gap (sidebar-to-content), card-to-card gap |
| 2xl | 48px | Login panel vertical padding |
| 3xl | 64px | Login page full-height centering offset |

Exceptions:
- Touch targets on action icon buttons (edit, delete in table rows): minimum 40px height — use `p-button-sm` with explicit min-height to meet usability standard.
- Sidebar width: 256px (fixed, not on 8-point scale — PrimeNG standard sidebar width).
- Header height: 56px (PrimeNG `p-toolbar` default — not adjusted).

## Typography

| Role | Size | Weight | Line Height | PrimeNG Class / CSS |
|------|------|--------|-------------|---------------------|
| Body | 14px | 400 (regular) | 1.5 | Default PrimeNG body; `font-size: 0.875rem` |
| Label | 12px | 600 (semibold) | 1.4 | Form field labels, table column headers; `font-size: 0.75rem; font-weight: 600` |
| Heading | 20px | 600 (semibold) | 1.3 | Page title in header, dialog title; `font-size: 1.25rem; font-weight: 600` |
| Display | 28px | 700 (bold) | 1.2 | Login page system name "Quản lý Kho Thiết Bị" on left panel; `font-size: 1.75rem; font-weight: 700` |

**Only 2 weights in use across Body/Label/Heading:** 400 and 600. Display (login only) uses 700 for emphasis.

**Source:** Claude's Discretion (CONTEXT.md). Standard admin UI scale — readable at typical enterprise monitor resolution (1080p+).

## Color

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) | `#F8F9FA` | App background, page content area, table row backgrounds |
| Secondary (30%) | `#FFFFFF` | Sidebar background, cards, dialog panels, header background |
| Accent (10%) | `#2563EB` | Primary CTA buttons only (Thêm mới, Lưu), active sidebar item indicator, pagination current-page highlight |
| Destructive | `#DC2626` | Delete action buttons and confirmation dialogs only |

Accent reserved for:
1. "Thêm mới" (Add) button on each reference data page — one per page
2. "Lưu" (Save) button inside Add/Edit modal dialogs — one per dialog
3. Active sidebar navigation item left-border indicator (3px left border)
4. Current page number button in table pagination
5. No other elements use accent color

Sidebar disabled items (unbuilt phases): text color `#9CA3AF` (gray-400), no hover state, `cursor: not-allowed`.

Login left panel background: `#1E3A5F` (deep navy) — creates visual contrast from the white right panel and establishes branding. System name and logo render in `#FFFFFF` on this panel.

**Source:** Claude's Discretion (CONTEXT.md — "color scheme / branding — để Claude chọn"). Professional admin tool palette using PrimeNG Lara Light Blue theme as base, overriding primary to `#2563EB`.

## Component Inventory

### 1. Login Page (`/login`)

**Layout:** Full-viewport split, two equal columns (50% / 50%).

Left panel:
- Background: `#1E3A5F`
- Content centered vertically: company logo placeholder (80x80px, white border, `#FFFFFF` text initials "KTB") + system name "Quản lý Kho Thiết Bị" at Display size + tagline "Hệ thống quản lý tài sản công ty" at Body size, color `#93C5FD`
- No interactive elements

Right panel:
- Background: `#FFFFFF`
- Content centered vertically, max-width 400px, horizontal padding 48px
- Form title: "Đăng nhập" at Heading size
- Fields: `p-inputtext` for Username, `p-password` for Password (toggle show/hide)
- Submit button: "Đăng nhập" — full width, `p-button` with accent color, height 44px
- Error message zone: below the submit button, rendered as `p-message severity="error"` — visible only when error state is active
- No "forgot password" link (out of scope, AUTH-V2-01 is v2)

Responsive: Below 768px, left panel collapses to hidden; right panel takes full width.

**Source:** CONTEXT.md D-12.

### 4. Reference Data Table Pages (`/categories`, `/suppliers`, `/departments`)

Each page follows the identical layout contract:

**Page header zone** (below app header, above table):
- Left: Page title (Heading size)
- Right: "Thêm mới" button (`p-button`, accent color, `pi-plus` icon prefix)

**Search zone** (between page header and table):
- `p-inputgroup` with `pi-search` prefix icon + `p-inputtext` placeholder "Tìm kiếm theo tên..."
- Width: 320px, left-aligned
- Filters table rows client-side by name on input (debounced 300ms)

**Table** (`p-table`):
- `[paginator]="true"` with `[rows]="20"` — server-side pagination
- `[lazy]="true"` — triggers `onLazyLoad` on page/sort change
- Columns per entity:

  Categories:
  | # | Tên danh mục | Mô tả | Thao tác |
  |---|---|---|---|

  Suppliers:
  | # | Tên nhà cung cấp | Địa chỉ | Thông tin liên hệ | Thao tác |
  |---|---|---|---|---|

  Departments:
  | # | Tên phòng ban | Thao tác |
  |---|---|---|

- "Thao tác" column: two icon buttons — Edit (`pi-pencil`, `p-button text sm`) + Delete (`pi-trash`, `p-button text sm danger/destructive`)
- Row hover: background `#F1F5F9`
- Paginator text: "{first} - {last} trong tổng số {totalRecords} bản ghi"
- Empty state: see Copywriting Contract

**Loading state:** `p-skeleton` rows (3 rows, matching column count) displayed while API call is in flight. No spinner overlay — inline skeleton preferred.

**Source:** CONTEXT.md D-18, D-19, D-20, D-21, D-27.

### 5. Add/Edit Modal Dialog

Used for all reference data create and update operations. Does not navigate to a new page.

`p-dialog`:
- `[modal]="true"`, `[draggable]="false"`, `[resizable]="false"`
- Width: 480px on desktop, 92vw on mobile
- Header: "Thêm danh mục" / "Sửa danh mục" / "Thêm nhà cung cấp" / etc. (Heading size)
- Footer: two buttons — "Hủy" (`p-button outlined secondary`, left) + "Lưu" (`p-button accent`, right)

Form layout inside dialog: single-column, 16px gap between fields, `p-floatlabel` for floating labels.

Fields per entity:

Categories: Tên danh mục (required, max 100), Mô tả (optional, textarea, max 500)
Suppliers: Tên nhà cung cấp (required, max 200), Địa chỉ (optional, max 500), Thông tin liên hệ (optional, max 200)
Departments: Tên phòng ban (required, max 100)

Validation display: `p-message severity="error"` beneath the invalid field — shown on blur and on submit attempt. Never show validation errors before the user touches the field.

**Source:** CONTEXT.md D-19.

### 6. Delete Confirmation

Triggered by clicking the delete icon in the table row.

`p-confirmDialog` (PrimeNG built-in):
- Header: "Xác nhận xóa"
- Message: "Bạn có chắc chắn muốn xóa [tên bản ghi]?"
- Accept button: "Xóa" — destructive color `#DC2626`
- Reject button: "Hủy" — outlined secondary

**Delete-blocked error (in-use records):** When the API returns a 409 Conflict (entity in use):
- Do NOT show the confirmDialog
- Show `p-toast severity="error"` positioned top-right
- Toast message: see Copywriting Contract
- Auto-dismiss: 6 seconds (longer than default 3s to allow reading the Vietnamese message)

**Source:** CONTEXT.md D-22, D-23.

### 7. Dashboard Placeholder (`/dashboard`)

Phase 1 delivers a placeholder page only. Full dashboard is Phase 6.

Content: centered card with `pi-home` icon (48px), heading "Dashboard", body text "Chức năng này sẽ được hoàn thiện ở giai đoạn sau." No action buttons.

**Source:** CONTEXT.md D-13.

## Interaction States

| State | Pattern |
|-------|---------|
| Loading (table data) | `p-skeleton` inline rows — no full-page spinner |
| Loading (dialog save) | "Lưu" button shows `pi-spinner` icon spinning + disabled; text changes to "Đang lưu..." |
| Loading (login submit) | "Đăng nhập" button shows `pi-spinner` + disabled; text changes to "Đang xử lý..." |
| Success feedback | `p-toast severity="success"` top-right, auto-dismiss 3s |
| Error feedback (API) | `p-toast severity="error"` top-right, auto-dismiss 6s |
| Error feedback (login) | `p-message severity="error"` inline below submit button, persistent until next submit attempt |
| Form invalid — blur | `p-message severity="error"` beneath invalid field |
| Form invalid — submit | All invalid fields show errors simultaneously; focus moves to first invalid field |
| Delete confirmation | `p-confirmDialog` modal — requires explicit user confirmation |
| Delete blocked | `p-toast severity="error"` — no confirmation dialog shown |
| Disabled nav item hover | No visual change; tooltip "Chức năng này chưa được triển khai." appears after 500ms |
